## 巴什博弈

> 有n个物品，两个人轮流从这堆物品中取物，规定每次至少取一个，最多取 *m* 个，最后取光者得胜。

* 结论：当取前的数值为 (*m*+1)∗*k* ,即 n%(m+1)==0  的时候为先手必败。其余的都是先手必胜。

我们可以自己尝试m从1开始手玩几下，就可以推出哪些状态是必胜/败的，画博弈图啥的，最后其实会发现是一个大的循环递推过程



## Nim和

[P2197 【模板】Nim 游戏 - 洛谷](https://www.luogu.com.cn/problem/P2197)

>  Nim 游戏的规则是这样的：地上有 *n* 堆石子，每人每次可从任意一堆石子里取出任意多枚石子扔掉，可以取完，不能不取。每次只能从一堆里取。最后没石子可取的人就输了。假如甲是先手，且告诉你这 *n* 堆石子的数量，他想知道是否存在先手必胜的策略。

* 结论：s1^s2^s3^s4^...^sn=0  时，先手必败，否则先手必胜

* 简单分析：
  * 当玩家面临的状态为异或0时，必败，我们现在可以从两堆石子开始分析，如果异或和不为0，那么后手的策略是：先手上一步选什么，后手随同
  * 当玩家面临的状态不为异或0时，他的每一步必定是把状态调至异或0
  * 因此，某种意义上，这是一个二循环

## 威佐夫博弈

> 有两堆石子,数量分别为 a,b(a<=b), 两人轮流取,
>
> * 可以在一堆中取任意多个
> * 在两堆中取相同多个
>
> 取走最后一个者获胜

结论:  令 k=b-a , phi= (sqrt(5.0)+1.0)/2.0 =1.618, 黄金分割数

* a=(sqrt(5.0)+1.0)*k/2.0 ,则后手胜,奇异局势
* 先手胜

https://www.luogu.com.cn/problem/P2252



```c++
void solve() {
    f80 phi=(sqrt(5.0l)+1.0l)/2.0l;
    i64 a,b;
    cin>>a>>b;
    if (a>b) swap(a,b);

    i64 k=b-a;
    i64 tp=(i64)( k*phi+1e-9);
    if (a==tp) cout<<"0\n";
    else cout<<"1\n";

}
```

# 字符串型

[Problem - 1527B1 - Codeforces](https://codeforces.com/problemset/problem/1527/B1)

> 简单版本和困难版本之间的唯一区别是，简单版本中给定的字符串 s最初是一个回文，对于困难版本，此条件并不总是成立，回文是一个从左到右和从右到左读取相同的字符串。例如，“101101”是回文，而“0101”不是。Alice和Bob正在玩一个字符串 s（在这个版本中最初是一个回文），长度为 n，由字符‘0’和‘1’组成的游戏。双方轮流上场，爱丽丝先走。在每个回合中，玩家可以执行以下操作之一：
>
> * 选择任意 i ( 1≤i≤n )，其中 s[i]=为‘0’，并将 s[i]更改为‘1’,支付1美元。
>
> * 把整个字符串反过来，支付0美元。此操作仅在字符串当前不是回文，并且最后一次操作不是反转时才允许。也就是说，如果Alice把字符串倒过来，那么Bob就不能在下一步中倒过来，反之亦然。
>
> 反转字符串意味着从最后到第一个字母重新排序。例如，“01001”反转后变成“10010”。
>
> 当字符串中的每个字符都变成“1”时，游戏结束。在此之前花费最少金钱的玩家赢得游戏，如果两人花费相同的金钱，则为平局。如果两个玩家都玩得很好，输出是Alice赢，Bob赢，还是平局。



* 每一步都想实现对对方的限制，我们只需关注cnt0的奇偶
